# v2.3.3 — 痛点与改进建议

## 一、用户真实痛点

### 1. 在线源失效率高，更新必须发版
- `assets/online_sources.json` 目前是**打包在 APK 内的本地文件**，通过 `rootBundle.loadString()` 加载
- 站点一死就必须等下一个版本，用户在此期间搜不到任何结果
- v2.3.2 的 CHANGELOG 写了"支持热更新"但实际并未实现远程拉取
- **建议**：从 `auth.congutsun.com/sources.json` 拉取最新站点列表，本地文件作为首次安装的兜底；设置页显示"在线源版本"和"上次更新时间"

### 2. 播放失败原因不明
- 当前播放失败统一显示"播放失败：xxx"或"未找到可播放的视频源"
- 用户不知道是网络问题（值得重试）、源已失效（换源即可）还是格式不支持（放弃）
- 视频加载超时后无任何提示，用户只能干等或退出
- **建议**：区分"源返回404/503""网络超时""视频格式不支持"三类错误 + 对应操作提示

### 3. 平板/大屏布局浪费空间
- 详情页、剧集列表在大屏上是单列全宽拉伸，400+dp 的侧边区域全是空白
- 播放器横屏时看剧集需要点顶部选集按钮 → 弹出半屏面板，不是侧边栏那种沉浸式体验
- **建议**：横屏播放时右侧固定剧集列表面板（当前是弹窗，改成常驻侧栏）；详情页宽度 > 720dp 时分栏布局

### 4. 下载完成无通知
- BT 下载任务完成后用户除非主动打开下载中心，否则不知道
- 下载卡在 99% 没有反馈
- **建议**：下载完成/失败发送系统通知栏消息；相应用户离开 App 后的场景

### 5. 看完番无后续引导
- 播放完一集后自动播下一集，但如果已经是最后一集，就停在播放结束画面
- 用户想看"我看了哪些""该追什么"需要回到首页自己找
- **建议**：播放完成 + 无下一集时，弹出"标记看过"+"去评分"快捷面板；首页加"最近观看"行

### 6. Bangumi 互动只读
- 吐槽/评论只能看，不能回复
- 想看一个番的评分需要去网页端操作
- **建议**：接入 Bangumi 发表评论 API；详情页加评分入口

### 7. HTTP 明文站点风险
- Dodo 动漫适配器使用 `http://m.dodoge.me`，中间人可注入恶意视频 URL
- **建议**：改为 `https://`，若不支持标记为降级

## 二、技术痛点

### 8. VideoPlayerPage 2350 行
- 一个文件承载播放控制、在线搜索、下载管理、PiP、弹幕、手势、自动下一集全部逻辑
- 改任何功能都需要在这个文件里翻找，回归风险高
- **建议**：先提取 ViewModel（状态逻辑），再拆 Controls/Danmaku/Gesture 为独立 Widget

### 9. 剩余 API 方法仍在吞异常
- 3 个方法（characters/persons/relations）已迁移到 `_cachedListGet` 泛型 helper
- 其余 13 个仍在用 `catch (e) { return []; }`，网络错误和空数据无法区分
- **建议**：逐步迁移剩余方法
